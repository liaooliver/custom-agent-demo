# Todo List 前端規格文件

## 1. 專案概述

### 1.1 專案目標
開發一個純前端的 Todo List 應用程式，提供基本的待辦事項管理功能（新增、讀取、更新、刪除）。

### 1.2 技術範圍
- **前端框架**：Angular
- **樣式**：CSS
- **資料儲存**：localStorage（瀏覽器本地儲存）
- **後端串接**：無（純前端應用）

---

## 2. 功能需求

### 2.1 新增 (Create)
- 使用者可以透過輸入欄位新增待辦事項
- **必填欄位**：
  - 標題（文字）
- **自動產生欄位**：
  - 寫入時間（timestamp）
  - 唯一識別碼（用於 CRUD 操作）
- **互動流程**：
  1. 使用者在輸入框輸入標題
  2. 按下「新增」按鈕或按 Enter 鍵
  3. 新待辦事項出現在列表中
  4. 輸入框清空，準備下次輸入

### 2.2 讀取 (Read)
- 顯示所有待辦事項列表
- **顯示資訊**：
  - 標題
  - 順序（依照寫入時間排序，最新的在最上方或最下方，需決定）
  - 寫入時間（格式：YYYY-MM-DD HH:mm:ss 或其他可讀格式）
- 頁面載入時自動從 localStorage 讀取資料並顯示

### 2.3 更新 (Update)
- 使用者可以編輯待辦事項的標題
- **互動流程**：
  1. 點擊待辦事項旁的「編輯」按鈕
  2. 標題欄位變為可編輯狀態（例如：顯示輸入框）
  3. 使用者修改標題
  4. 按下「儲存」按鈕或按 Enter 鍵確認修改
  5. 更新後的標題顯示在列表中
  6. 若按下「取消」按鈕或 ESC 鍵則取消編輯
- **注意**：寫入時間不變，僅更新標題

### 2.4 刪除 (Delete)
- 使用者可以刪除待辦事項
- **互動流程**：
  1. 點擊待辦事項旁的「刪除」按鈕
  2. 彈出確認對話框（confirm dialog）
  3. 確認對話框內容：「確定要刪除此待辦事項嗎？」
  4. 若使用者點擊「確定」，則從列表和 localStorage 中移除該項目
  5. 若使用者點擊「取消」，則關閉對話框，不執行刪除

## 3. 資料結構

### 3.1 Todo Item 物件結構
```typescript
interface TodoItem {
  id: string;          // 唯一識別碼 (UUID 或 timestamp)
  title: string;       // 標題
  createdAt: number;   // 寫入時間 (timestamp)
}
```

### 3.2 localStorage 儲存格式
- **Key**: `todoList`
- **Value**: JSON 字串，包含 TodoItem 陣列
```json
[
  {
    "id": "1701234567890",
    "title": "買牛奶",
    "createdAt": 1701234567890
  },
  {
    "id": "1701234567891",
    "title": "寫報告",
    "createdAt": 1701234567891
  }
]
```

## 4. UI 元件規劃

### 4.1 主要元件
1. **輸入區域**
   - 標題輸入框（input text）
   - 新增按鈕（button）

2. **待辦事項列表**
   - 每個待辦事項包含：
     - 標題顯示區
     - 寫入時間顯示區
     - 編輯按鈕
     - 刪除按鈕

3. **編輯模式**（inline editing）
   - 標題輸入框（編輯時出現）
   - 儲存按鈕
   - 取消按鈕

4. **刪除確認對話框**
   - 提示訊息
   - 確定按鈕
   - 取消按鈕

### 4.2 UI 要求
- **響應式設計**：不需要
- **設計風格**：無特定要求（可採用簡潔實用風格）

### 5.2 LocalStorage 操作
- **讀取**：`localStorage.getItem('todoList')`
- **寫入**：`localStorage.setItem('todoList', JSON.stringify(data))`
- **清除**：`localStorage.removeItem('todoList')`（若需要）

### 5.3 排序邏輯
- 預設按照 `createdAt` 時間戳排序
- 建議：最新的項目顯示在最上方（降序排列）
- 可在實作時調整排序方向

## 6. 使用者互動流程

### 6.1 完整操作流程
1. **首次進入**
   - 檢查 localStorage 是否有資料
   - 若無資料，顯示空列表
   - 若有資料，載入並顯示

2. **新增待辦事項**
   - 輸入標題 → 點擊新增 → 儲存至 localStorage → 更新畫面

3. **編輯待辦事項**
   - 點擊編輯 → 進入編輯模式 → 修改標題 → 儲存 → 更新 localStorage → 更新畫面

4. **刪除待辦事項**
   - 點擊刪除 → 彈出確認對話框 → 確認 → 從 localStorage 移除 → 更新畫面

## 8. 補充說明

### 8.1 錯誤處理
- 若標題為空，不允許新增
- localStorage 存取失敗時的處理（例如：瀏覽器隱私模式）

### 8.2 未來擴充建議（不在此次範圍內）
- 完成/未完成狀態切換
- 待辦事項優先順序
- 搜尋/篩選功能
- 標籤分類

### 8.3 瀏覽器支援
- 支援現代瀏覽器（Chrome, Firefox, Safari, Edge）
- 需支援 localStorage API
